# Phase 3.6: Piper Subprocess TTS + Three-Source Fallback Chain — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the third `ITTSSource` implementation: `PiperTTSSource` spawns a bundled [Piper](https://github.com/rhasspy/piper) CLI binary as a subprocess, pipes text to stdin, and reads raw PCM from stdout. With three sources in place, wire the fallback chain from spec §6.5 so a scene tagged `tts.source = "piper"` with `tts.fallback = "apple"` automatically falls back to AVSpeechSynthesizer if Piper synthesis fails — and ultimately to prebaked if both live sources fail.

**Architecture:** `PiperTTSSource` uses `juce::ChildProcess` to run a Piper binary on raw-PCM mode (`--output-raw`). Piper voice models (`.onnx` + `.onnx.json`) are bundled into `Contents/Resources/piper/voices/` via the existing CMake post-build copy. Source selection in `PluginProcessor` is extended: on synthesis failure, it walks the `tts.fallback` chain declared in scene JSON. The TTSPrewarmer is generalized to support multiple sources (one prewarmer per source); the active scene's source decides which prewarmer to query.

**Tech Stack:** C++20, JUCE 8 (`ChildProcess`, `MemoryBlock`), Catch2 v3 — no new external library dependencies. The Piper binary itself (a single ~30 MB executable) and one voice model (~63 MB) are vendored as binary blobs under `Resources/piper/`.

**Reference spec:** [`docs/superpowers/specs/2026-05-29-while-my-guitar-gently-speaks-design.md`](../specs/2026-05-29-while-my-guitar-gently-speaks-design.md) §6.2 (three sources, Piper row), §6.4 (prewarming), §6.5 (failure handling with fallback chain), §7.2 (scene `tts.fallback`).

**Pre-flight assumption:** Phase 3.5 is merged into `main` (Apple TTS + TTSPrewarmer in place, 76/76 tests pass, scene 6 uses Apple). The pending follow-up chips (type-and-say UI, hardening) are independent and can land in any order relative to this phase.

If you're new to this codebase, read these first:

- **Spec §6.2 row "PiperTTSSource"** for the design intent: subprocess + bundled binary, ~0.3–0.8 s synthesis latency, voice model from disk.
- **Existing `src/audio/PrebakedTTSSource.{h,cpp}` and `src/audio/AppleTTSSource.{h,mm}`** as reference implementations of `ITTSSource`. `PiperTTSSource` follows the same shape (`prepare(sr)` + `synthesize(key)`).
- **Existing `src/audio/TTSPrewarmer.{h,cpp}`** — Phase 3.5 wired it for AppleTTSSource only. Phase 3.6 generalizes: per-source prewarmers managed in PluginProcessor.
- **`src/app/PluginProcessor.cpp` lines ~110–145** — the existing TTS source routing in the scene-change callAsync. Phase 3.6 extends this to walk the fallback chain.
- **Spec §6.5 "failure handling"** for the fallback semantics: if the primary source fails (returns nullptr), try the source named in `tts.fallback`; recurse through chain until success or end.
- **Piper CLI docs**: [https://github.com/rhasspy/piper/blob/master/docs/CLI.md](https://github.com/rhasspy/piper). Key invocation:
  - `piper --model voice.onnx --output-raw < text.txt > audio.raw`
  - Output is mono Int16 PCM at the voice's native rate (typically 22050 Hz).

**Threading rules (unchanged):**
- Audio thread: no allocations, no locks. PiperTTSSource is message-thread-only.
- Per-source prewarmers each spawn one background thread; safe.

**What Phase 3.6 deliberately does NOT do**:
- No karaoke text (Phase 5).
- No type-and-say UI (separate follow-up chip).
- No Piper voice picker UI — voice is configured via scene JSON only.
- No carousel DSP (Phase 4).

```cpp
#pragma once

#include <string>

#include "ITTSSource.h"

namespace guitar_dsp::audio {

// Live TTS source using the bundled Piper CLI binary. `synthesize(text)`
// spawns the binary as a subprocess, pipes text to stdin, reads raw
// mono Int16 PCM from stdout, and packages the result as a TTSClipPtr.
//
// Latency is typically 300-800 ms on Apple Silicon — slower than
// AppleTTSSource for short phrases, faster for long ones; voice quality
// is generally crisper than Apple's compact voices.
//
// Returns nullptr if:
//   - the binary doesn't exist or isn't executable
//   - the voice model doesn't exist
//   - the subprocess exits non-zero or times out
//   - the stdout buffer is empty
//
// Call from the message thread, NOT the audio thread. Pair with
// TTSPrewarmer to hide latency at scene activation.
class PiperTTSSource : public ITTSSource {
public:
    // binaryPath: full path to the `piper` executable.
    // voicePath:  full path to the .onnx voice model. The matching
    //             .onnx.json must sit next to it (Piper's convention).
    PiperTTSSource(std::string binaryPath, std::string voicePath);

    void prepare(double targetSampleRate);

    // For PiperTTSSource, `key` is the text to synthesize.
    TTSClipPtr synthesize(const std::string& key) override;
    std::string sourceName() const override { return "piper"; }

    // Returns true if both the binary and voice exist on disk and the
    // binary is executable. Use this for diagnostic UI / startup checks.
    bool isReady() const;

private:
    std::string binaryPath_;
    std::string voicePath_;
    double      targetSampleRate_ = 48000.0;
};

} // namespace guitar_dsp::audio
```

## Task 2: PiperTTSSource implementation — subprocess + PCM read

**Files:**

`juce::ChildProcess` provides `start()`, `writeToStdin()`, `readAllProcessOutput()`. The blocking pattern: spawn → write text → close stdin → block on stdout read with timeout → parse Int16 PCM.

- [ ] **Step 1: Write `src/audio/PiperTTSSource.cpp`**

```cpp
#include "PiperTTSSource.h"

#include <juce_core/juce_core.h>

#include <algorithm>
#include <cmath>
#include <filesystem>
#include <iostream>

namespace guitar_dsp::audio {

namespace {

// Convert interleaved/mono Int16 PCM to a float vector at the given
// source rate, then linear-resample to the target rate (mono only).
std::vector<float> int16PcmToFloatMono(const std::byte* bytes,
                                       std::size_t numBytes,
                                       double srcRate,
                                       double targetRate) {
    const std::size_t numSamples = numBytes / sizeof(std::int16_t);
    const auto* src = reinterpret_cast<const std::int16_t*>(bytes);

    std::vector<float> floatSrc(numSamples);
    constexpr float invInt16 = 1.0f / 32768.0f;
    for (std::size_t i = 0; i < numSamples; ++i) {
        floatSrc[i] = static_cast<float>(src[i]) * invInt16;
    }

    if (std::abs(srcRate - targetRate) < 0.5) {
        return floatSrc;
    }
    // Linear resample.
    const double ratio = srcRate / targetRate;
    const std::size_t outLen = static_cast<std::size_t>(numSamples / ratio);
    std::vector<float> out(outLen);
    for (std::size_t i = 0; i < outLen; ++i) {
        const double srcIdx = i * ratio;
        const std::size_t i0 = static_cast<std::size_t>(srcIdx);
        const float frac = static_cast<float>(srcIdx - i0);
        const std::size_t i1 = std::min(i0 + 1, numSamples - 1);
        out[i] = (1.0f - frac) * floatSrc[i0] + frac * floatSrc[i1];
    }
    return out;
}

// Piper voice .onnx.json contains a "sample_rate" field. Parse it; on
// failure, return 22050 (Piper's most common default).
double readVoiceSampleRate(const std::string& voicePath) {
    namespace fs = std::filesystem;
    const fs::path jsonPath = fs::path(voicePath).string() + ".json";
    juce::File jf(jsonPath.string());
    if (!jf.existsAsFile()) return 22050.0;
    const auto text = jf.loadFileAsString();
    auto parsed = juce::JSON::parse(text);
    if (auto* obj = parsed.getDynamicObject()) {
        if (obj->hasProperty("audio")) {
            if (auto* audio = obj->getProperty("audio").getDynamicObject()) {
                if (audio->hasProperty("sample_rate")) {
                    return static_cast<double>(audio->getProperty("sample_rate"));
                }
            }
        }
        if (obj->hasProperty("sample_rate")) {
            return static_cast<double>(obj->getProperty("sample_rate"));
        }
    }
    return 22050.0;
}

} // namespace

PiperTTSSource::PiperTTSSource(std::string binaryPath, std::string voicePath)
    : binaryPath_(std::move(binaryPath)), voicePath_(std::move(voicePath)) {}

void PiperTTSSource::prepare(double targetSampleRate) {
    targetSampleRate_ = targetSampleRate;
}

bool PiperTTSSource::isReady() const {
    namespace fs = std::filesystem;
    if (!fs::exists(binaryPath_)) return false;
    if (!fs::exists(voicePath_)) return false;
    // Check executable bit on the binary.
    const auto perms = fs::status(binaryPath_).permissions();
    return (perms & (fs::perms::owner_exec | fs::perms::group_exec |
                     fs::perms::others_exec)) != fs::perms::none;
}

TTSClipPtr PiperTTSSource::synthesize(const std::string& text) {
    if (text.empty()) return nullptr;
    if (!isReady()) {
        std::cerr << "[PiperTTSSource] not ready (binary or voice missing): "
                  << binaryPath_ << " / " << voicePath_ << '\n';
        return nullptr;
    }

    juce::StringArray argv;
    argv.add(juce::String(binaryPath_));
    argv.add("--model");
    argv.add(juce::String(voicePath_));
    argv.add("--output-raw");

    juce::ChildProcess proc;
    if (!proc.start(argv, juce::ChildProcess::wantStdOut)) {
        std::cerr << "[PiperTTSSource] failed to start subprocess\n";
        return nullptr;
    }

    // Pipe the text + close stdin.
    proc.writeToStdin(juce::String(text) + "\n");
    proc.closeStdin();

    // Read all stdout. Piper exits when synthesis is done.
    juce::MemoryBlock stdoutBuf;
    constexpr int kTimeoutMs = 10000;
    const auto t0 = juce::Time::getMillisecondCounterHiRes();
    juce::MemoryBlock chunk;
    chunk.setSize(8192);
    while (proc.isRunning() ||
           proc.readProcessOutput(chunk.getData(), 0) > 0 /* drain */) {
        const auto n = proc.readProcessOutput(chunk.getData(),
                                              static_cast<int>(chunk.getSize()));
        if (n > 0) stdoutBuf.append(chunk.getData(), static_cast<std::size_t>(n));
        const auto elapsed = juce::Time::getMillisecondCounterHiRes() - t0;
        if (elapsed > kTimeoutMs) {
            std::cerr << "[PiperTTSSource] timeout, killing subprocess\n";
            proc.kill();
            return nullptr;
        }
        if (!proc.isRunning() && n == 0) break;
    }

    if (stdoutBuf.getSize() == 0) {
        std::cerr << "[PiperTTSSource] no audio from subprocess\n";
        return nullptr;
    }

    const double srcRate = readVoiceSampleRate(voicePath_);
    auto samples = int16PcmToFloatMono(
        static_cast<const std::byte*>(stdoutBuf.getData()),
        stdoutBuf.getSize(),
        srcRate,
        targetSampleRate_);

    auto clip = std::make_shared<TTSClip>();
    clip->name = text.substr(0, 32);
    clip->sampleRate = targetSampleRate_;
    clip->samples = std::move(samples);
    return clip;
}

} // namespace guitar_dsp::audio
```

```bash
#!/usr/bin/env bash
# Downloads the Piper TTS binary and the en_US-amy-medium voice model
# into assets/piper/. Idempotent — skips files that already exist.
#
# Usage: ./scripts/fetch_piper.sh
#
# Requires: curl, tar. Optionally: shasum for verification.

set -euo pipefail

cd "$(dirname "$0")/.."

PIPER_DIR="assets/piper"
PIPER_BIN="$PIPER_DIR/piper"
VOICE_ONNX="$PIPER_DIR/voices/en_US-amy-medium.onnx"
VOICE_JSON="$PIPER_DIR/voices/en_US-amy-medium.onnx.json"

# Pick the right Piper release for this platform.
case "$(uname -s)/$(uname -m)" in
    Darwin/arm64)
        PIPER_URL="https://github.com/rhasspy/piper/releases/download/v1.2.0/piper_macos_aarch64.tar.gz"
        ;;
    Darwin/x86_64)
        PIPER_URL="https://github.com/rhasspy/piper/releases/download/v1.2.0/piper_macos_x64.tar.gz"
        ;;
    *)
        echo "Unsupported platform: $(uname -s)/$(uname -m)" >&2
        exit 1
        ;;
esac

VOICE_BASE="https://huggingface.co/rhasspy/piper-voices/resolve/main/en/en_US/amy/medium"

mkdir -p "$PIPER_DIR/voices"

if [ ! -x "$PIPER_BIN" ]; then
    echo "Fetching Piper binary..."
    tmp=$(mktemp -d)
    curl -L --fail -o "$tmp/piper.tar.gz" "$PIPER_URL"
    tar -xzf "$tmp/piper.tar.gz" -C "$tmp"
    # The release tarball contains a "piper/" directory with the binary
    # and shared libraries. Copy everything into assets/piper/.
    cp -R "$tmp/piper/." "$PIPER_DIR/"
    chmod +x "$PIPER_BIN"
    rm -rf "$tmp"
    echo "  Installed: $PIPER_BIN"
else
    echo "Piper binary already present: $PIPER_BIN"
fi

if [ ! -f "$VOICE_ONNX" ]; then
    echo "Fetching en_US-amy-medium voice..."
    curl -L --fail -o "$VOICE_ONNX"  "$VOICE_BASE/en_US-amy-medium.onnx"
    curl -L --fail -o "$VOICE_JSON" "$VOICE_BASE/en_US-amy-medium.onnx.json"
    echo "  Installed: $VOICE_ONNX"
else
    echo "Voice already present: $VOICE_ONNX"
fi

echo "Done. The next build will copy these into the .app bundle."
```

- [ ] **Step 1: Refactor the chain logic into a free function**

Define a small struct for clarity:

```cpp
namespace {

// Map of source-name -> ITTSSource*. The map is owned by PluginProcessor;
// this just provides lookup.
using TTSSourceRegistry = std::map<std::string, audio::ITTSSource*>;

// Read the cfg's source, try synthesize; on nullptr, recurse via fallback
// until success or chain exhaustion. Per source, `key` is what each
// source's synthesize() expects (clip name for prebaked, text for live).
// `keyFor(name)` returns the key to use for a given source-name (handles
// the prebaked-uses-clip / live-uses-text distinction).
audio::TTSClipPtr synthesizeWithFallback(
    const scenes::TtsConfig& cfg,
    const TTSSourceRegistry& registry,
    const std::function<std::string(const std::string&)>& keyFor,
    int maxDepth = 4)
{
    std::string sourceName = cfg.source;
    std::string fallback   = cfg.fallback;

    for (int depth = 0; depth < maxDepth; ++depth) {
        if (sourceName.empty()) break;
        auto it = registry.find(sourceName);
        if (it == registry.end() || it->second == nullptr) {
            // Source not available; skip to fallback.
        } else {
            const std::string key = keyFor(sourceName);
            if (!key.empty()) {
                if (auto clip = it->second->synthesize(key)) return clip;
            }
        }
        // Failed; advance to fallback. Fallback has no further chain in
        // this design — keep it simple. If you want chains > 2 deep,
        // load the fallback's TtsConfig from somewhere; out of scope for
        // Phase 3.6.
        if (fallback == sourceName) break;  // self-loop guard
        sourceName = fallback;
        fallback.clear();  // chain terminates after one fallback hop
    }
    return nullptr;
}

} // namespace
```
